LocationShell — Scoped Loading States

Works with your existing architecture. No restructuring needed.
Suspense boundaries inside LocationShell give each section its own skeleton independently.

Your Architecture + Where Loading Goes

LocationShell (server component — re-renders on every nav)
  │
  ├── <Suspense fallback=<BreadcrumbSkeleton/>>
  │     └── Breadcrumbs          ← shows skeleton while resolving path
  │
  ├── <Suspense fallback=<FunctionsNavSkeleton/>>
  │     └── FunctionsNav         ← shows skeleton while fetching functions
  │
  └── <Suspense fallback=<ContentSkeleton/>>  ← Col3 only
        └── children             ← page content (plans, tickets etc.)

Each section loads independently — breadcrumbs can resolve
before functions are ready. Content shows last.

FunctionsNav uses useTransition — spinner on clicked item,
no full re-render of LocationShell.
💡
Why Suspense inside LocationShell? Since LocationShell is a server component that wraps everything, loading.tsx files would replace ALL of it. Suspense lets each child section stream independently — breadcrumbs appear first, then nav, then content. Zero layout shifts.
✅
Non-breaking changes only. We add skeleton components + wrap existing renders in Suspense. LocationShell's logic, Shell component, and all imports stay exactly as you have them. Only additions — nothing removed.

LocationShell.tsx — Add Suspense Boundaries Key ChangeServer

Your existing logic stays 100% intact. Only change: wrap each section in Suspense with skeleton fallback. Split data fetching into separate async components so Suspense can stream them independently.

⚠️
Important pattern: For Suspense to work on server components, the async data fetch must happen inside a child component, not in the parent. We extract FunctionsNavLoader as a separate async component so its Suspense boundary fires correctly.
src/components/layout/LocationShell.tsx 
import { Suspense } from 'react';
import { notFound, redirect } from 'next/navigation';
import { getLocationsTree } from
  '@/modules/locations/actions/server-locations';
import { getLocationFunctions } from
  '@/modules/locations/actions/server-location-functions';
import { FunctionsNav } from '../nav/FunctionsNav';
import { FUNCTION_LABELS } from
  '@/modules/locations/lib/constants';
import { Breadcrumbs } from './Breadcrumbs';
import { NoFunctionsAvailable } from
  '../nav/NoFunctionsAvailable';
import {
  getNodeAndPathBySlug,
  locationUrl,
} from '@/modules/locations/lib/utils';

// ── NEW skeleton imports ──────────────────────────────
import { BreadcrumbSkeleton } from
  './BreadcrumbSkeleton';
import { FunctionsNavSkeleton } from
  '../nav/FunctionsNavSkeleton';
import { ContentSkeleton } from
  './ContentSkeleton';

interface Props {
  slug: string;
  activeFn: string | null;
  breadcrumbs: BreadcrumbItem[];
  children: React.ReactNode;
}

// ── NEW: Separate async component for functions data ──
// This allows Suspense to independently stream FunctionsNav
// while breadcrumbs and content may already be visible.
async function FunctionsNavLoader({
  locationId,
  locationSlug,
  activeFn,
}: {
  locationId: number;
  locationSlug: string;
  activeFn: string | null;
}) {
  const functions = await getLocationFunctions(locationId);

  if (functions.length === 0) {
    return <NoFunctionsAvailable />;
  }

  return (
    <FunctionsNav
      locationSlug={locationSlug}
      activeFn={activeFn}
      functions={functions}
    />
  );
}

// ── LocationShell — your existing component, Suspense added ──
export async function LocationShell({
  slug,
  activeFn,
  children,
}: {
  slug: string;
  activeFn: string | null;
  children: React.ReactNode;
}) {
  // ── YOUR EXISTING LOGIC — completely unchanged ────────
  const tree = await getLocationsTree();
  const found = getNodeAndPathBySlug(tree, slug);
  if (!found) notFound();

  const { node, path } = found;
  const breadcrumbs = buildBreadcrumbs(node, path, activeFn);

  return (
    {/* YOUR existing Shell wrapper — unchanged */}
    <Shell
      slug={slug}
      activeFn={activeFn}
      breadcrumbs={breadcrumbs}
    >
      {/* ── NEW: Suspense around FunctionsNavLoader ── */}
      <Shell.Nav>
        <Suspense fallback={<FunctionsNavSkeleton />}>
          <FunctionsNavLoader
            locationId={node.id}
            locationSlug={slug}
            activeFn={activeFn}
          />
        </Suspense>
      </Shell.Nav>

      {/* ── NEW: Suspense around Col3 content ── */}
      <Shell.Content>
        <Suspense fallback={<ContentSkeleton />}>
          {children}
        </Suspense>
      </Shell.Content>
    </Shell>
  );
}

  breadcrumbs,

FunctionsNav.tsx — useTransition on click ClientUpdated

Spinner on clicked item during transition. Prevents double-click. Dims all items while navigating.

⚠️
Replace your existing <Link> with this button pattern. Your current code uses Link — this triggers LocationShell to re-render fully. useTransition + router.push only re-renders the [fn] page segment, leaving breadcrumbs and nav visible.
src/components/nav/FunctionsNav.tsx 
'use client';

import { useState, useTransition } from 'react';
import { useRouter, usePathname } from 'next/navigation';
import type { LocationFunction } from '@/types/locations';

interface Props {
  locationSlug: string;
  activeFn: string | null;
  functions: LocationFunction[];
}

export function FunctionsNav({
  locationSlug,
  activeFn,
  functions,
}: Props) {
  const router = useRouter();
  const pathname = usePathname();
  const [isPending, startTransition] = useTransition();
  const [pendingSlug, setPendingSlug] =
    useState<string | null>(null);

  // Clear pending when transition completes
  if (!isPending && pendingSlug) setPendingSlug(null);

  function handleClick(fnSlug: string) {
    const href = `/location/${locationSlug}/${fnSlug}`;
    if (pathname === href || isPending) return;
    setPendingSlug(fnSlug);
    startTransition(() => router.push(href));
  }

  return (
    <nav className="py-2">
      {functions.map((fn) => {
        const href = `/location/${locationSlug}/${fn.slug}`;
        const isActive = pathname === href;
        const isThisPending =
          isPending && pendingSlug === fn.slug;

        return (
          <button
            key={fn.slug}
            type="button"
            onClick={() => handleClick(fn.slug)}
            disabled={isPending}
            aria-current={isActive ? 'page' : undefined}
            className={[
              'flex w-full items-center justify-between',
              'px-4 py-2.5 text-sm text-left',
              'border-l-2 transition-colors',
              isActive
                ? 'border-blue-600 bg-blue-50 font-medium text-blue-700'
                : 'border-transparent text-gray-600'
                  + ' hover:bg-gray-50 hover:text-gray-900',
              isPending
                ? 'opacity-50 cursor-not-allowed'
                : 'cursor-pointer',
            ].join(' ')}
          >
            <span>{fn.label}</span>
            {isThisPending && (
              <svg
                className="h-3.5 w-3.5 animate-spin text-blue-500"
                viewBox="0 0 24 24" fill="none"
              >
                <circle cx="12" cy="12" r="10"
                  stroke="currentColor" strokeWidth="4"
                  className="opacity-25"/>
                <path fill="currentColor"
                  className="opacity-75"
                  d="M4 12a8 8 0 018-8V0C5.373 0 0 5.373 0 12h4z"/>
              </svg>
            )}
          </button>
        );
      })}
    </nav>
  );
}
